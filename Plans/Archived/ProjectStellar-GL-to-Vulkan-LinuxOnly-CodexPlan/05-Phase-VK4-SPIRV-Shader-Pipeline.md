# Phase VK-4 — SPIR-V Shader Pipeline

## Objective

Create the Vulkan shader build pipeline and initial graphics pipeline objects needed for static level rendering.

## Files To Add

```text
assets/shaders/vulkan/static_level.vert
assets/shaders/vulkan/static_level.frag
src/graphics/vulkan/VulkanShaderModules.cpp
include/stellar/graphics/vulkan/VulkanShaderModules.hpp
```

Avoid making shader helpers public unless needed by tests.

## Shader Source Direction

Use GLSL targeting Vulkan/SPIR-V:

- Explicit `layout(location = N)` vertex attributes.
- Explicit `layout(set = S, binding = B)` descriptors.
- UBO or push constants for draw uniforms.
- Texture slots aligned with current material contract.

Required vertex inputs from existing `StaticVertex` contract:

| Location | Attribute |
|---:|---|
| 0 | position |
| 1 | normal |
| 2 | uv0 |
| 3 | tangent |
| 4 | uv1 |
| 5 | color |

## CMake Shader Compilation

When `STELLAR_ENABLE_VULKAN_BACKEND=ON`, compile shaders with Vulkan SDK tooling if available.

Preferred:

```cmake
find_program(GLSLC_EXECUTABLE glslc HINTS "$ENV{VULKAN_SDK}/bin")
```

Then add custom commands that generate:

```text
${CMAKE_BINARY_DIR}/generated/shaders/vulkan/static_level.vert.spv
${CMAKE_BINARY_DIR}/generated/shaders/vulkan/static_level.frag.spv
```

If `glslc` is unavailable, fail configuration only for Vulkan builds with a clear message.

## Shader Packaging

Prefer one of these:

1. Generate C++ byte arrays from SPIR-V for deterministic tests.
2. Copy `.spv` files beside the executable and load from build-time path.

For Codex simplicity, start with runtime loading from build output, but keep a helper so generated byte arrays can replace it later.

## Pipeline Objects

Add creation for:

- descriptor set layout
- pipeline layout
- render pass or dynamic rendering setup
- opaque graphics pipeline
- alpha-blend graphics pipeline

Alpha-mask can initially share the opaque pipeline and use shader discard.

## Tests

Add a display-free shader test:

```text
tests/graphics/VulkanShaderCompile.cpp
```

It should confirm:

- SPIR-V files exist.
- Shader modules can be created against a Vulkan device when opt-in GPU context is available, or skipped with code 77.
- For pure display-free validation, at least verify CMake generated the expected files.

## Validation

```bash
cmake --preset linux-vulkan
cmake --build --preset linux-vulkan --parallel $(nproc)
ctest --test-dir build-linux-vulkan -R 'vulkan_shader' --output-on-failure
```

## Acceptance Criteria

- Vulkan-enabled build compiles shader sources to SPIR-V.
- Pipeline creation is possible in opt-in Vulkan smoke path.
- No shader or Vulkan tooling is required for macOS Metal builds.
- Metal shader tests remain independent.
