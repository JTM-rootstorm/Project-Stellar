# Phase 5E Vulkan Render Parity Breakdown

## Goal

Make `--renderer vulkan` truthful for glTF runtime rendering by replacing the current upload-record/no-op Vulkan backend with an incremental presented renderer that reaches parity with the existing OpenGL lightweight material path.

## Current State

- `VulkanGraphicsDevice` initializes a Vulkan instance, SDL surface, physical device, logical device, and graphics/present queue.
- `create_mesh`, `create_texture`, and `create_material` validate and retain backend-neutral records only.
- `begin_frame`, `draw_mesh`, and `end_frame` are explicit no-ops.
- OpenGL parity target includes `StaticVertex` attributes, base color, vertex color, UV0/UV1 routing, alpha mask/blend, double-sided culling, normal mapping, metallic/roughness factors and texture, occlusion, emissive, and uniform-array skinning capped at 96 joints.
- Default tests must remain display-free; GPU/context tests should be opt-in.

## Suggested Chunking

### Phase 5E.1: Swapchain And Clear-Only Presentation

Goal: make `begin_frame` and `end_frame` acquire, clear, submit, and present a frame without mesh drawing.

Tasks:

2. Create swapchain, swapchain images, image views, and per-image framebuffers or dynamic-rendering equivalents.
3. Create a render pass or dynamic rendering path with color and depth attachments.
4. Add depth image allocation and view creation.
5. Add frame acquisition, queue submission, presentation, and resize/out-of-date handling skeleton.
6. Ensure `destroy_vulkan_objects` tears down swapchain resources before device/instance resources.
7. Keep `draw_mesh` harmless while no graphics pipeline exists.

Validation:

- Existing default tests unchanged.
- Optional Vulkan context test executes a clear/present frame when a Vulkan-capable display environment is explicitly enabled.
- Manual `stellar-client --renderer vulkan` should show a black/clear window rather than a no-op window if display is available.

Completion notes (2026-04-27):

  `VulkanGraphicsDevice`: device initialization now creates a swapchain, swapchain image views,

### Phase 5E.2: Command Pools, Per-Frame Resources, And Synchronization

Goal: stabilize frame lifetime before resource uploads and draw calls grow.

Tasks:

1. Add command pool for the selected graphics queue family.
2. Add per-frame command buffers, image-available semaphores, render-finished semaphores, and in-flight fences.
3. Track a small fixed number of frames in flight.
4. Reset/re-record command buffers per frame.
5. Handle `VK_ERROR_OUT_OF_DATE_KHR` and resize by recreating swapchain-dependent resources.
6. Avoid blocking every frame except where required for safe teardown or resize.

Validation:

- Optional clear/present smoke test runs multiple frames.
- Manual window resize does not crash when using `--renderer vulkan`.

### Phase 5E.3: Static Mesh GPU Buffers

Goal: turn `create_mesh` from record-only into real vertex/index GPU upload for the existing `StaticVertex` layout.

Tasks:

1. Add Vulkan buffer allocation helpers for vertex, index, uniform/staging buffers.
2. Choose an initial memory strategy: direct host-visible uploads are acceptable for the first slice; staging/device-local can follow if needed.
3. Upload each mesh primitive's `StaticVertex` array and `uint32_t` index array.
4. Store per-primitive buffer handles, index count, and feature flags.
5. Destroy mesh buffers in `destroy_mesh` and backend teardown.
6. Add CPU-side tests for vertex input metadata if helpers can be isolated without a Vulkan device.

Validation:

- Existing `RenderScene` upload/inspection tests remain green.
- Optional Vulkan upload smoke test creates representative fixtures without drawing if a Vulkan device is available.

### Phase 5E.4: Minimal Graphics Pipeline And Solid Material Draw

Goal: make Vulkan draw opaque static geometry with transforms and base color.

Tasks:

1. Add shader build strategy for Vulkan SPIR-V. Recommended: commit source GLSL/HLSL and generated SPIR-V only if the repo does not already require a shader compiler at build time; otherwise add an explicit CMake shader compile path.
2. Create descriptor layout for per-draw transforms and material factors.
3. Create pipeline layout, graphics pipeline, viewport/scissor setup, depth test, and back-face culling defaults.
4. Bind vertex/index buffers and issue indexed draws from `draw_mesh`.
5. Support fallback material color matching OpenGL when material handle is missing.
6. Initially handle only opaque, non-textured, non-skinned primitives.

Validation:

- Optional Vulkan render smoke test draws the debug cube or a generated triangle.
- Manual `stellar-client --renderer vulkan` shows a rotating cube with all faces present.

### Phase 5E.5: Texture Images, Image Views, And Samplers

Goal: implement `create_texture` and sampler creation for backend-neutral texture state.

Tasks:

1. Map image formats and color spaces to Vulkan formats: `R8G8B8A8_SRGB`, `R8G8B8A8_UNORM`, and any required RGB handling strategy.
2. Upload images through staging buffers or host-visible images if kept deliberately simple.
3. Create image views for sampled textures.
4. Create Vulkan samplers from `SamplerAsset`: min/mag filters, mip filters, wrap modes.
5. Generate or handle mip levels consistently with OpenGL's current behavior.
6. Destroy image, view, memory, and sampler resources in `destroy_texture`.

Validation:

- Optional Vulkan texture smoke test uploads generated white/normal fixtures.
- Manual textured glTF scene renders with base color texture.

### Phase 5E.6: Material Descriptor Routing And Lightweight Shader Parity

Goal: port OpenGL's lightweight material shader behavior into Vulkan for non-skinned static draws.

Tasks:

1. Add descriptor sets or push constants for material factors and feature flags.
2. Bind base color, normal, metallic-roughness, occlusion, and emissive textures with safe default textures when absent.
3. Implement UV0/UV1 routing per texture slot.
4. Implement vertex color multiplication.
5. Implement alpha mask discard.
6. Implement normal map shading when primitive tangents are present.
7. Implement metallic/roughness, occlusion, and emissive behavior matching the OpenGL shader, not full glTF PBR.

Validation:

- Use Phase 5D representative fixtures as the test asset source.
- Optional Vulkan render smoke tests can target deterministic basics such as clear color and simple textured/material output.
- Manual representative static glTF run with `--renderer vulkan`.

### Phase 5E.7: Alpha Blend, Double-Sided Culling, And Draw-State Batching

Goal: match OpenGL visible ordering and raster state for alpha and culling.

Tasks:

1. Add pipeline variants or dynamic state for opaque/mask vs blend and culling vs double-sided.
2. Enable blend with `srcAlpha`/`oneMinusSrcAlpha` for alpha blend materials.
3. Disable culling for double-sided materials and keep back-face culling otherwise.
4. Preserve `RenderScene`'s existing opaque/mask/front-to-back and blend/back-to-front submission assumptions.
5. Avoid changing backend-neutral draw ordering unless a Vulkan-specific issue proves it necessary.

Validation:

- Existing display-free ordering tests remain green.
- Optional Vulkan alpha smoke manually/visually validates alpha mask and blend fixtures.

### Phase 5E.8: Skinning Descriptor Path

Goal: render skinned glTF primitives in Vulkan with the same current limits as OpenGL.

Tasks:

1. Add joint matrix storage for draw calls, initially matching OpenGL's 96-joint cap unless a broader buffer model is intentionally chosen.
2. Add shader support for `joints0` and `weights0` attributes.
3. Bind identity/no-skin path for static primitives and joint matrices for skinned primitives.
4. Validate over-limit skin palettes fail or fall back clearly rather than reading invalid descriptor data.
5. Keep larger skin palette expansion for Phase 5F unless explicitly pulled forward.

Validation:

- Existing animation/runtime and `RenderScene` skin pose forwarding tests remain green.
- Optional Vulkan skinned fixture smoke test.
- Manual animated/skinned glTF run with `--renderer vulkan`.

### Phase 5E.9: Resize, Recreate, And Resource Lifetime Hardening

Goal: make the Vulkan renderer robust enough for normal client use.

Tasks:

1. Recreate swapchain-dependent resources on window resize, suboptimal swapchain, or out-of-date presentation.
2. Ensure in-flight resource lifetime is safe when meshes/textures/materials are destroyed.
3. Wait for device idle only on teardown or unavoidable resize paths.
4. Audit all Vulkan calls for clear error reporting through `std::expected` during initialization/upload paths.
5. Keep frame functions `noexcept`; log or safely no-op after unrecoverable frame errors.

Validation:

- Manual resize test with static cube and representative glTF.
- Optional Vulkan smoke test that recreates swapchain if feasible in hidden-window environment.

## Key Decisions To Keep Explicit

- Shader toolchain: choose between checked-in SPIR-V or a CMake-time compiler dependency before Phase 5E.4.
- Memory allocation: start with simple internal Vulkan allocation; introduce VMA or a larger allocator only if manual allocation becomes a blocker.
- Vulkan API style: choose render pass/framebuffer vs dynamic rendering based on target Vulkan 1.2 constraints and driver support.
- Skinning limit: match OpenGL's current 96-joint limit for Phase 5E; defer larger palettes to Phase 5F unless required.
- Testing: never make GPU/display Vulkan tests part of default CTest.
